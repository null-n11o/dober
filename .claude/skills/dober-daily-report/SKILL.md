# dober-daily-report

## Skill Description

Doberの日次メモリーを管理する。セッション末の「日報に追記して」で作業ログを蓄積し、1日の終わりに「日報をまとめて」で整理・MEMORY.md更新まで行う。

## Triggers

### トリガーA：追記モード
- 「日報に追記して」
- 「日報に記録して」
- 「セッション終了」

### トリガーB：まとめモード
- 「日報をまとめて」
- 「今日の日報を完成させて」

---

## トリガーA：追記モード

セッション内で行ったことを `memory/daily/YYYY-MM-DD.md` に追記する。
**MEMORY.md は更新しない。**

### Step 1: 今日の日報ファイルを確認する

`memory/daily/YYYY-MM-DD.md` が存在するか確認する。
- 存在する → そのまま追記
- 存在しない → 以下のテンプレートで新規作成してから追記

```markdown
# 日報 YYYY-MM-DD

<!-- このファイルはセッションごとに追記される。「日報をまとめて」で整理する。 -->
```

### Step 2: セッションの内容を追記する

ファイルの末尾に以下の形式で追記する:

```markdown
## セッション HH:MM

### やったこと
- [具体的な作業1]
- [具体的な作業2]

### 決定・気づき
- [重要な決定や発見があれば]

### 次にやること
- [ ] [継続タスク]
```

---

## トリガーB：まとめモード

その日の全追記内容を読み込み、整理・要約して `memory/daily/YYYY-MM-DD.md` を上書きする。
さらに `ROADMAP.md` と `memory/MEMORY.md` を更新する。

### Step 1: 今日の日報を読み込んで整理する

全セッションの追記内容をまとめ、以下の形式に整理して上書き保存する:

```markdown
# 日報 YYYY-MM-DD

## 今日やったこと
- [全セッションの作業を統合・重複削除]

## 決定事項
- [今日の重要な意思決定]

## 気づき・学び
- [パターンや発見]

## 明日やること
- [ ] [優先タスク1]
- [ ] [優先タスク2]

## 数値（変化があれば）
- Threadsフォロワー: X人
- メールリスト: X人
```

### Step 2: 重要な決定があれば decisions/ に記録する

方針変更・戦略転換・新ルール追加・ローンチ設計の確定があった場合のみ `memory/decisions/YYYY-MM-DD-xxx.md` を作成する。

### Step 3: ROADMAP.md のタスクステータスを更新する

- 完了したタスクを ✅ 完了に移動する
- 新たに着手したタスクを 🟡 進行中に移動する
- 新しいタスクが発生していれば 🔴 未着手に追加する

### Step 4: MEMORY.md を更新する

以下の場合に該当セクションを更新する:
- KPIに変化 → 「KPI スナップショット」を更新
- 新しいパターン発見 → 「蓄積パターン」に追記
- フォーカスが変わった → 「現在のフェーズ」を更新
- 重要な決定があれば → 「直近の重要決定」テーブルに追加

### 出力フォーマット

```
✅ 日報 YYYY-MM-DD をまとめました
今日の要点（3行以内）
明日の優先タスク TOP3
```
