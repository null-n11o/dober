# Dober 記事アウトライン生成スキル

テーマと種別（LM / note）を受け取り、Doberブランドに沿った章構成を生成する。
このアウトラインは後続の `dober-article-search` と `dober-article-write` が使用する。

## 前提

作業開始前に必ず参照:
- `assets/brand-guideline.md` — ボイス・トーン・構造

## インプット

| 項目 | 必須 | 説明 |
|------|:----:|------|
| テーマ | ○ | 記事の中心テーマ（例:「男の総合値」「資産1000万」） |
| 種別 | ○ | `LM` または `note` |
| ターゲットペルソナ | △ | 省略時はDoberデフォルト（20〜30代男性） |

不足していたらユーザーに確認する。

## 実行手順

### Step 1: テーマ分析

- テーマをDoberの5領域（精神・恋愛・経済・知力・身体）のどれに分類するか確認する
- ターゲットが抱える「現状の痛み」と「理想の状態」を定義する

### Step 2: 全体構成の設計

**LMの場合（構成パターン）:**
```
導入: 読者の現状・痛みを突きつける（共感ではなく診断）
第1章〜第N章: 原因分析 → 原則提示 → 実践手順
終章: 変化後の自分像と次のアクション
```

**noteの場合（構成パターン）:**
```
導入: フックとなる問い・現実の突きつけ
第1章〜第N章: 論点ごとに分解、SEOクエリに対応した章タイトル
終章: 行動指示で締める
```

### Step 3: アウトライン出力

以下のフォーマットで出力する:

```markdown
# [仮タイトル]

## 基本情報
- 種別: LM / note
- 対象: [ターゲット読者]
- 中心テーマ: [テーマ]
- 想定文字数: [総文字数]

## 章構成

### 導入（〜XXX字）
- 目的: [この章で達成すること]
- キーポイント:
  - [ポイント1]
  - [ポイント2]
- トーン: 🔴最強 / 🟠強 / 🟡中 / 🟢低

### 第1章: [章タイトル]（〜XXX字）
- 目的: [この章で達成すること]
- キーポイント:
  - [ポイント1]
  - [ポイント2]
  - [ポイント3]
- トーン: 🔴最強 / 🟠強 / 🟡中 / 🟢低
- 要リサーチ: [Webサーチが必要なポイント、なければ「なし」]

（第N章まで繰り返す）

### 終章: [章タイトル]（〜XXX字）
- 目的: [この章で達成すること]
- キーポイント:
  - [ポイント1]
- トーン: 🟡中 / 🟢低
- CTA: [読者に促す次の行動]
```

### Step 4: 確認

出力後、ユーザーに以下を確認する:
- 章の追加・削除・順序変更はあるか
- 各章のトーンレベルは適切か
- このアウトラインで `dober-article-search` → `dober-article-write` に進んでよいか

## トーンレベルの使い分け（brand-guideline.md より）

| レベル | 場面 |
|--------|------|
| 🔴最強 | 怠惰・言い訳・逃避を描写する場面 |
| 🟠強 | リスク・失敗パターンを語る場面 |
| 🟡中 | 解決策・行動リストを示す場面 |
| 🟢低 | 真実・哲学を静かに語る場面 |

## エラーハンドリング

| 状況 | 対処 |
|------|------|
| テーマが広すぎる | 「どの側面に絞るか」をユーザーに確認する |
| 種別が不明 | LM か note かを確認してから進む |
| 想定文字数が10,000字を大幅に超える | 章数を絞るか記事を分割することを提案する |
