# Dober タイトル生成スキル

執筆済みの記事・LMを受け取り、Doberブランドのタイトル候補を複数生成する。
タイトルは必ず本文執筆後に作る（内容に合わせるため）。

## 前提

作業開始前に必ず参照:
- `assets/brand-guideline.md` — フックの公式・高インプレッションパターン

## インプット

| 項目 | 必須 | 説明 |
|------|:----:|------|
| 記事の概要 or 全文 | ○ | 本文全体 or 各章タイトル + キーポイントのサマリー |
| 種別 | ○ | `LM` または `note` |
| ターゲットキーワード | △ | noteの場合のみ。SEOで狙うキーワード |

## 実行手順

### Step 1: 記事の本質を把握する

本文から以下を抽出する:
- **中心的な約束**: この記事を読むと読者は何を得られるか
- **ターゲットの痛み**: 読者が抱える最大の問題・不安
- **記事のユニークな切り口**: 他にない視点・フレームワーク・実例

### Step 2: タイトル候補を生成する

brand-guideline.mdの**高インプレッションフックパターン**を参照し、種別に応じて候補を生成する。

**LMタイトルの方針:**
- 「読むと何が手に入るか」を明示する（ベネフィット直球型）
- ターゲット（20〜30代男性）が「俺のことだ」と感じる言葉を使う
- シリーズ化を意識する場合は「① ②」など番号を付ける

**noteタイトルの方針:**
- ターゲットキーワードを自然に含める（SEO）
- 検索意図（知りたい・解決したい・比較したい）に対応する
- クリックされるフックパターンを使いつつ検索クエリと一致させる

**使うフックパターン（brand-guideline.md より）:**

| パターン | テンプレート |
|---------|------------|
| 年齢×致命的ミス | `[年齢]で犯す、[N]つの致命的な[X]` |
| 禁止リスト | `絶対に[行動]してはいけない[対象]` |
| やめる×やる | `[対象]の「やめる」ことと「やる」こと` |
| ベネフィット直球 | `[ターゲット]が[理想の状態]になるための[方法]` |
| 逆説・意外性 | `[一見ネガティブ]が[実は最強の理由]` |
| 数字リスト | `[対象]が知るべき[N]つの[X]` |
| 対比リバース | `[世間の常識] vs [本当の真実]` |

### Step 3: 候補の出力

以下のフォーマットで出力する:

```markdown
## タイトル候補

### 推奨（最もクリックされる可能性が高い）
**[タイトル]**
- 採用パターン: [フックパターン名]
- 理由: [なぜこれが刺さるか]

### 候補A
**[タイトル]**
- 採用パターン: [フックパターン名]
- 特徴: [推奨との違い・使い分け]

### 候補B
**[タイトル]**
- 採用パターン: [フックパターン名]
- 特徴: [推奨との違い・使い分け]

### 候補C（note専用: SEO重視）
**[タイトル]**
- 含むキーワード: [ターゲットキーワード]
- 特徴: 検索流入を優先した構成

---
### サブタイトル案（LMのみ）
- [サブタイトル案1]
- [サブタイトル案2]
```

### Step 4: 選定のサポート

候補を出した後、以下を添える:

- **A/Bテスト推奨ペア**: クリック率を比較するなら「推奨 vs 候補X」
- **使い分けの基準**: LMであれば配布先（メルマガ/note/SNS）によって変える場合の指針

## チェックリスト

出力前に各候補を確認する:
- [ ] 20〜30代男性が「自分ごと」と感じる言葉が入っているか
- [ ] 「何が得られるか」または「何を避けられるか」が伝わるか
- [ ] エモジ・ハッシュタグが入っていないか
- [ ] 35字以内に収まっているか（長すぎると切れる）
- [ ] noteの場合、ターゲットキーワードが含まれているか

## エラーハンドリング

| 状況 | 対処 |
|------|------|
| 記事の概要が不明瞭 | 各章タイトルだけでも渡してもらい、そこから生成する |
| noteのキーワードが未指定 | 本文の内容から想定キーワードを提案し、確認してから生成する |
| 「どれも刺さらない」と言われた | 本文の最も強い一文・事例を抽出し、それをタイトルに転用する |
