# Dober 記事リサーチスキル

アウトラインの1セクション分を受け取り、Webサーチで情報を収集する。
1セクション = 1回の実行。収集した情報は `dober-article-write` に渡す。

## 前提

- `dober-article-outline` で生成したアウトラインの1章分を入力として受け取る
- アウトラインの「要リサーチ」が「なし」の章はこのスキルをスキップしてよい

## インプット

| 項目 | 必須 | 説明 |
|------|:----:|------|
| セクション情報 | ○ | アウトラインの対象章（タイトル・キーポイント・要リサーチ） |
| テーマ・文脈 | ○ | 記事全体のテーマ（検索クエリの精度を上げるため） |

## 実行手順

### Step 1: 検索クエリの設計

アウトラインの「要リサーチ」と「キーポイント」からWebサーチクエリを設計する。

**クエリ設計の方針:**
- 1セクションにつき2〜4クエリを設計する
- 統計・研究・事例を狙う（権威づけに使えるデータを優先）
- 日本語クエリと英語クエリを混在させると情報量が増える

**クエリの種類:**
| 種類 | 例 |
|------|-----|
| 統計・数値 | 「○○ 統計 日本 2024」 |
| 研究・エビデンス | 「○○ 研究 効果 論文」 |
| 事例・具体例 | 「○○ 成功事例 具体例」 |
| 定義・解説 | 「○○ とは メカニズム」 |

### Step 2: Webサーチ実行

設計したクエリでWebサーチを実行する。

- 各クエリの上位3件程度の情報を収集する
- 信頼性の低いソース（まとめサイト・個人ブログ）より公的機関・研究機関・専門メディアを優先する

### Step 3: 情報の整理・選別

収集した情報を以下の基準で選別する:

**採用する情報:**
- 数字・統計（具体性が増す）
- 権威ある引用元（研究・専門家・歴史的事実）
- Doberの主張を裏付けるエビデンス
- 読者の「現実の痛み」を示すデータ

**採用しない情報:**
- 曖昧な一般論（Doberのボイスと相性が悪い）
- ネガティブなDoberブランドと矛盾する内容
- 古すぎるデータ（5年以上前は要注意）

### Step 4: リサーチ結果の出力

以下のフォーマットで出力する:

```markdown
## リサーチ結果: [セクション名]

### 収集した主要情報

**[テーマA]**
- [事実・データ・引用]（出典: ○○）
- [事実・データ・引用]（出典: ○○）

**[テーマB]**
- [事実・データ・引用]（出典: ○○）

### 執筆に使えるエビデンス（優先度順）

1. [最も強力なデータ・事実] — 使用推奨箇所: [キーポイントX]
2. [次に強力なデータ・事実] — 使用推奨箇所: [キーポイントY]
3. [補足として使えるデータ] — 使用推奨箇所: [キーポイントZ]

### リサーチで判明した追加インサイト

- [アウトラインに追加すべき観点があれば記載。なければ「なし」]

### dober-article-write への引き継ぎメモ

- トーン: [アウトライン指定のトーンレベル]
- 推奨する数字・統計: [執筆中に必ず使うデータ]
- 避けるべき表現: [収集情報の中でDoberブランドと相性が悪いもの]
```

## エラーハンドリング

| 状況 | 対処 |
|------|------|
| 有効な情報が見つからない | クエリを変えて再検索。それでもなければ「要リサーチ: 情報不足」と明記してwriteスキルに引き継ぐ |
| 矛盾する情報が複数ある | 最も信頼性の高いソースを採用し、採用理由をメモに記載 |
| 「要リサーチ: なし」の章 | このスキルをスキップし、アウトラインの情報のみでdober-article-writeに進む |
