# Dober — 事業コンテキスト

> このファイルはDoberに関わる全てのAgentが作業開始前に必ず読む。

---

## セッション開始時の必須手順

新しいセッションが始まったら、最初に以下のファイルを読む:

1. **`memory/MEMORY.md`**
   — KPIスナップショット・蓄積パターン・優先タスクを把握する

2. **必要に応じて読む:**
   - `memory/daily/` の直近3日分 — 最近の作業コンテキスト

3. **セッション終了時（または「日報を作って」と言われたら）:**
   - `/dober-daily-report` スキルを実行してメモリーを更新する

> このメモリーシステムがDoberの「育つAI」の核心。毎セッション読み書きを徹底する。

## 事業概要

| 項目 | 内容 |
|------|------|
| 事業名 | Dober |
| 事業種別 | 自社プロダクト（個人ブランド × コンテンツビジネス） |
| ステータス | 運営中（マネタイズ実装フェーズ） |
| 概要 | 20〜30代男性向けに「精神・恋愛・経済・知力・身体」の5領域で発信するコンテンツブランド。Threadsをメイン集客源とし、リードマグネット→メールリスト→有料商品のファネルを構築中。 |

---

## ターゲット顧客

**プライマリーペルソナ**
- 属性: 20〜30代男性、現状に閉塞感を持つ会社員・フリーランス
- 課題・ニーズ: 恋愛・金・健康・自己規律のいずれかに不満。甘えを認識しており、変わりたいが方法論がない。
- 行動特性: SNS（特にThreads/X）でインプット中毒気味。共感より「喝」を求めている。

**セカンダリーペルソナ**
- 副業・資産形成に関心はあるが行動できていない男性

## マネタイズファネル

```
Threads投稿（フック）← n8n自動投稿 ✅
  ↓ リプ欄にメルマガ登録の導線を置く（3時間以内に1000インプレッションを超えたら）
メルマガLPへ遷移
  ↓
メールリスト登録（現在: 68人）
  ↓
有料商品（設計中）
```

## 実装ステップ（優先順）

1. **LM-A / LM-B 作成** — カテゴリ別リストイン導線の拡充（LM-C=既存LM-0で代替可）
2. **n8n キーワードDM分岐実装** — 「評価」「資産」「規律」でLMを振り分け
3. **有料商品の設計・制作** — 恋愛OS + 金融OS + 規律ボーナスのバンドル
4. **Notion DBカテゴリタグ付け** — 全投稿に親/子カテゴリを付与
5. **高パフォーマンス投稿のリサイクル** — High Impressionをストックし定期再投稿
